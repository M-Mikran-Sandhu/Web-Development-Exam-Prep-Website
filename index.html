Q1 - React Basics & Fundamentals

25 Marks

What is React?

React is a JavaScript library created by Facebook for building user interfaces. It helps you build fast, interactive websites by automatically updating only the parts that changed.

üí° Example: Hello World Component
// Creating your first React component
1 import React from 'react'; // Import React library
2
3 function App() { // Create a function component
4   return <h1>Hello World!</h1>; // Return JSX
5 } // End of component
6
7 export default App; // Export component to use elsewhere

JSX - Write HTML in JavaScript

JSX looks like HTML but it's actually JavaScript. React converts it to JavaScript automatically. This makes writing UI code much easier.

üí° Example: JSX Syntax
// Understanding JSX
1 // This is JSX - looks like HTML
2 const element = <h1>Hello</h1>; // JSX syntax
3
4 // React converts it to this JavaScript
5 const element = React.createElement('h1', null, 'Hello'); // JavaScript

Components - Building Blocks

Components are like LEGO blocks. You create small pieces and combine them to build bigger things. Each component is reusable and independent.

üí° Example: Reusable Button Component
// Creating reusable components
1 function Button() { // Create Button component
2   return <button>Click Me</button>; // Return button JSX
3 } // End Button component
4
5 function App() { // Create App component
6   return ( // Return multiple components
7     <> // Fragment - wrapper without div
8       <Button /> // Use Button component
9       <Button /> // Reuse Button component
10     </> // End Fragment
11   ); // End return
12 } // End App component

Virtual DOM - How React Works

React keeps a copy of your webpage in memory called Virtual DOM. When something changes, it updates this copy first, then updates only the parts that changed on the real webpage. This is very fast!

üîë Key Concept: Virtual DOM ‚Üí Compare ‚Üí Update Real DOM ‚Üí Browser Shows Update

Application Rendering Sequence

Step 1: React reads your code
Step 2: Creates Virtual DOM (copy of webpage)
Step 3: Compares with real DOM
Step 4: Updates only what changed
Step 5: Browser shows the update

Props - Passing Data to Components

Props are like function parameters. You pass data from a parent component to a child component. Props are READ-ONLY - you cannot change them inside the component.

üí° Example: Using Props
// Passing data with props
1 function Greeting(props) { // Receive props as parameter
2   return <h1>Hello {props.name}!</h1>; // Use props.name
3 } // End component
4
5 function App() { // Parent component
6   return <Greeting name="John" />; // Pass name prop
7 } // End App

Props Destructuring

Instead of using props.name, you can extract the values directly. This is cleaner and more readable!

üí° Example: Destructuring Props
// Cleaner way to use props
1 function Greeting({ name, age }) { // Extract name and age directly
2   return <h1>{name} is {age} years old</h1>; // Use directly
3 } // End component
4
5 <Greeting name="John" age={25} /> // Pass multiple props

Component Types

Function Components: Simple JavaScript functions that return JSX. Most common today.
Class Components: Old way using ES6 classes. Less common now but still used.

üí° Example: Function Component (Modern)
// Modern way - Function component
1 function Welcome() { // Simple function
2   return <h1>Welcome!</h1>; // Return JSX
3 } // End component

Rendering Components

To use a component, you render it like an HTML tag. Component names MUST start with a capital letter!

A: Props are read-only data from parent to child. State is data that can change inside a component.

A: Use useEffect to fetch data, set up timers, or run code after the component renders.

Q: How do you handle forms in React?
A: Use useState to track input values. This creates a "controlled component".

Q: What are keys in lists?
A: Keys help React identify which items changed. Always use unique IDs, never use array index.
